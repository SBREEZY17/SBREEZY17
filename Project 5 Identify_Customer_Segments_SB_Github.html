Sarah Brown . September 2023

Introduction¶

In this project, I apply unsupervised learning techniques to identify segments of the German population that form the core customer base for a mail-order sales company in Germany. These segments can then be used to direct marketing campaigns towards audiences that will have the highest expected rate of returns.

The data that I use has been provided by our partners at Bertelsmann Arvato Analytics.

Step 0: Load the Data¶

There are four files associated with this project:

  • Udacity_AZDIAS_Subset.csv: Demographics data for the general population of Germany; 891211 persons (rows) x 85 features (columns).
  • Udacity_CUSTOMERS_Subset.csv: Demographics data for customers of a mail-order company; 191652 persons (rows) x 85 features (columns).
  • Data_Dictionary.md: Detailed information file about the features in the provided datasets.
  • AZDIAS_Feature_Summary.csv: Summary of feature attributes for demographics data; 85 features (rows) x 4 columns

Each row of the demographics files represents a single person, but also includes information outside of individuals, including information about their household, building, and neighborhood. I will use this information to cluster the general population into groups with similar demographic properties. Then, I will see how the people in the customers dataset fit into those created clusters.

The hope here is that certain clusters are over-represented in the customers data, as compared to the general population; those over-represented clusters will be assumed to be part of the core userbase. This information can then be used for further applications, such as targeting for a marketing campaign.

Discussion 1.2.2: Engineer Mixed-Type Features¶

Firstly I treated the suggested mixed variables of CAMEO_INTL_2015 and PRAEGENDE_JUGENDJAHRE.

With CAMEO_INTL_2015 it was a simple matter of splitting the two digits, which already had assigned significance in the dictionary. I gave them new column names and then deleted the original column.

For PRAEGENDE_JUGENDJAHRE, as several responses belonged to the same decade, I assigned a grouping number for each decade and then assigned a 1 or 0 to mainstream or not mainstream (by default, avantgarde). I gave them new column names and then deleted the original column.

In each case I made sure to treat the NaN carefully, so that each new column would have NaN in the same place.

For the other mixed variables, I looked them up in the dictionary for the definitions.

I concluded that none of the other mixed type variables were useful to keep.

LP_LEBENSPHASE_FEIN is simply a more detailed version of LP_LEBENSPAHSE_GROS, so deleting it would only cause us to loose some level of granularity.LP_LEBENSPAHSE_GROS is also not needed, because we just created a 'life_stage' column in the above engineering.

Wohnlage is a categorisation of neighbourhood quality. However we have no understanding of how this categorising is done, so the data loses relevance.

Finally PLZ8_BAUMAX refers to the most common building type within PL28 (which I assume is an area). This data is very specific and I don't believe it would contribute a great deal to the overall picture which is at country level.

Therefore, I dropped those columns as well.

What does PCA do?¶

Principal component analysis, or PCA, is a dimensionality reduction method that is often used to reduce the dimensionality of large data sets, by transforming a large set of variables into a smaller one that still contains most of the information in the large set.

The code projects the original data, which is four-dimensional, into less dimensions. After dimensionality reduction, there usually isnt a particular meaning assigned to each principal component. The new components are just the two main dimensions of variation.

Reducing the number of variables of a data set naturally comes at the expense of accuracy, but the trick in dimensionality reduction is to trade a little accuracy for simplicity. Smaller data sets are easier to explore and visualize and make analyzing data points much easier and faster for machine learning algorithms without extraneous variables to process.

PCA is often used to speed up the fitting of machine learning algorithms. It can also be used for data visualisation.

Step 2.3: Interpret Principal Components¶

Now that that data has transformed principal components, I will check out the weight of each variable on the first few components to see if they can be interpreted in some fashion.

As a reminder, each principal component is a unit vector that points in the direction of highest variance (after accounting for the variance captured by earlier principal components). The further a weight is from zero, the more the principal component is in the direction of the corresponding feature.

If two features have large weights of the same sign (both positive or both negative), then increases in one tend expect to be associated with increases in the other. To contrast, features with different signs can be expected to show a negative correlation: increases in one variable should result in a decrease in the other.

Step 3: Clustering¶

Step 3.1: Apply Clustering to General Population¶

I assessed and cleaned the demographics data, then scaled and transformed it. Now, it's time to see how the data clusters in the principal components space.

In this substep, I will apply k-means clustering to the dataset and use the average within-cluster distances from each point to their assigned cluster's centroid to decide on a number of clusters to keep.

I will do the following steps:

  • Use sklearn's KMeans class to perform k-means clustering on the PCA-transformed data.
  • Then, compute the average difference from each point to its assigned cluster's center.
  • Perform the above two steps for a number of different cluster counts. I will then see how the average distance decreases with an increasing number of clusters. However, each additional cluster provides a smaller net benefit. I will use this fact to select a final number of clusters in which to group the data.
  • Once I've selected a final number of clusters to use, I will re-fit a KMeans instance to perform the clustering operation, taking care to obtain the cluster assignments for the general demographics data, since I'll be using them in the final Step 3.3.

Step 3.3: Compare Customer Data to Demographics Data¶

At this point, I have clustered data based on demographics of the general population of Germany, and seen how the customer data for a mail-order sales company maps onto those demographic clusters. In this final substep, I will compare the two cluster distributions to see where the strongest customer base for the company is.

I will consider the proportion of persons in each cluster for the general population, and the proportions for the customers.

If the assumption is that the company's customer base to be universal, then the cluster assignment proportions should be fairly similar between the two.

If there are only particular segments of the population that are interested in the company's products, then I should see a mismatch from one to the other.

If there is a higher proportion of persons in a cluster for the customer data compared to the general population (e.g. 5% of persons are assigned to a cluster for the general population, but 15% of the customer data is closest to that cluster's centroid) then that suggests the people in that cluster should be a target audience for the company.

On the other hand, the proportion of the data in a cluster being larger in the general population than the customer data (e.g. only 2% of customers closest to a population centroid that captures 6% of the data) suggests that group of persons to be outside of the target demographics.

I will do the following points in this step:

Discussion 3.3: Compare Customer Data to Demographics Data¶

Conclusion¶

Those consumers represented by cluster 7 that should be targeted (as they are over represented) are largely positively explained by multi person households with multi generations in the same household. I deduce that these people have family responsibilities and therefore have less time to shop physically - but there may be many other reasons.

Those not interested in finance and low income earners being part of the most under represented cluster makes sense, as it would be avisable for the company not to target people with these characteristics.

It is also logical that the most over represented cluster and the most under represented cluster would both be well explained by component 0 (as it explains most of the variance of all the components) but negatively weighted by cluster 7 and positively weighted by cluster 8.

If the company would like to reinforce it's current strategy it should pay attention to cluster 7, but also to investigatecluster 2, 6 and 9 to get more granularity.

Limitations and opportunities for improvement¶

Despite being able to glean some information from the clusters which I analysed, I do suggest that it would be better to analyse more, if not all of the clusters to get a better picture of the target consumers. Cluster 7 suggested that multi person households would be within the group to target, but this seems almost common sense. It would be interesting to see more of the abstract features like energy consumption or shopper type coming up.

I encountered a problem at the customer cleaning stage when I applied the onehotencoder then was missing a column in the customer dataset, because '5' didn't appear as a response of GEBAEUDETYP in the customer data. I applied a workaround by creating a column and filling it with 0, however, to avoid this in future it could be improved by using a different type of encoder like leave one out coding or conditional probability encoding or target encoding.
